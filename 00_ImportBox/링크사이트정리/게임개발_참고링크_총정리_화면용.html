How to use

이 자료 사용법 · 분류 기준

긁어온 링크가 너무 많아서 그대로 두면 막상 필요할 때 못 찾아요. 그래서 두 가지 기준으로 정리했어요. 하나는 "무엇에 쓰는가"(엔진·에셋·사운드·UI·포켓몬…), 또 하나는 "어떤 형태인가"(실행 프로그램·웹사이트·GitHub 소스) 예요.

카드에 붙는 태그 읽는 법

각 링크 카드 위쪽에는 작은 태그가 붙어 있어요. 색만 보고도 "이게 깔아야 하는 프로그램인지, 그냥 보는 사이트인지"를 구분할 수 있게 한 거예요.

설치/도구 GitHub 소스 공식문서 참고/구경 무료 유료/혼합 저작권 주의
🧭읽는 순서 추천

1번 섹션(무조건 설치)부터 보세요. 지금 뼈대 잡는 단계에서 진짜 필요한 건 거기 5~6개가 전부예요. 나머지 섹션은 "맵 만들 차례", "소리 넣을 차례"처럼 그 작업을 할 때 꺼내 보는 사전처럼 쓰면 돼요.

1
Must install

⭐ 무조건 설치/가입해야 할 것

이 게임은 Phaser 3(2D 게임 도구 모음) + TypeScript(언어) + VS Code(편집기) 조합으로 만들기로 했죠. 이 조합을 돌리려면 아래 다섯 가지가 "있어야 시작이 되는" 필수품이에요. 나머지 100개 링크보다 이게 먼저예요.

①
Node.js엔진실 · 1순위
②
VS Code코드 편집기
③
Phaser 3게임 도구 모음
④
Git + GitHub세이브·백업
⑤
Tiled맵 그리기(곧 필요)
💡Node.js · VS Code · GitHub 계정은 직접 검색해서 받으세요

이 세 개는 공식 다운로드 페이지에서 본인 OS(Windows/Mac)에 맞는 설치 파일을 받아야 해서, 안전하게 각 공식 사이트(nodejs.org, code.visualstudio.com, github.com)에서 직접 받는 걸 권해요. 아래 카드들은 당신이 긁어온 목록에 실제로 들어있던, 바로 연결되는 사이트 위주로 정리했어요.

🎮Phaser 공식 사이트
설치/도구공식문서무료
우리가 쓰기로 한 2D 웹 게임 도구 모음. 다운로드·튜토리얼·문서가 여기 다 있어요. 지금 단계의 핵심 사이트.
이렇게 쓰기 → 시작 가이드(Getting Started) 따라 npm으로 설치, 예제 보며 씬·캐릭터 이동 익히기.
phaser.io 공식 GitHub 예제 모음
🗺️Tiled Map Editor
설치/도구무료
2D 타일맵(격자 맵) 만드는 무료 프로그램. 포켓몬·젤다식 맵 만들 때 사실상 표준이에요. Phaser가 Tiled 맵 파일을 바로 읽어요.
이렇게 쓰기 → "① 맵 이동" 만들 차례가 되면 설치. 맵을 그려서 내보내면 Phaser에 불러옴.
mapeditor.org GitHub
🎨Aseprite
설치/도구유료
도트(픽셀) 그림을 직접 그리는 가장 유명한 프로그램. 캐릭터·가구·아이템 직접 그릴 때 편해요. 유료지만 도트 게임에서 많이 씀.
이렇게 쓰기 → 처음엔 무료 에셋(4번 섹션)으로 시작하고, "내 그림"이 필요해지면 구매 고려.
aseprite.org
✏️Piskel
설치/도구무료
설치 없이 웹에서 바로 도트 스프라이트를 그릴 수 있는 무료 툴. Aseprite를 살지 고민될 때 먼저 써보기 좋아요.
이렇게 쓰기 → 간단한 캐릭터/아이콘을 웹에서 그려 PNG로 저장 → Phaser에 넣기.
piskelapp.com
✅설치 순서 한 줄 요약

Node.js → VS Code → GitHub 가입 → Phaser 설치(npm) 순서면 충분해요. Tiled·Piskel은 맵/그림 작업 들어갈 때 추가하면 돼요.

게임 엔진 · 개발 시작

게임 엔진은 "게임을 만드는 큰 틀"이에요. 당신은 이미 Phaser로 정했지만, 나중에 비교하거나 다른 길이 궁금할 때를 위해 모아둔 다른 엔진들도 정리했어요. 지금은 Phaser만 집중하면 됩니다.

"① 맵 이동"과 "도트 그림"을 만들 차례가 되면 쓰는 도구들이에요. 포켓몬식 게임은 결국 타일(격자) 맵 + 도트 캐릭터라서 이 두 종류 도구가 핵심이에요.

포켓몬 데이터 · 스프라이트

포켓몬류 게임의 핵심 자료예요. 데이터(이름·타입·능력치)를 어떻게 구조화할지, 이미지(스프라이트)를 어떻게 연결할지 배우는 데 씁니다. 단, 진짜 포켓몬 이미지/데이터를 그대로 배포하면 저작권 문제가 생기니, "구조·연습용"으로 보는 게 안전해요.

🧩PokéAPI가 데이터 공부의 중심

포켓몬 이름·타입·기술·능력치·이미지 URL을 API로 주기 때문에, "도감 데이터를 어떤 JSON 구조로 만들까"의 정답지가 돼요.

PokéAPI가 주는 데이터를 단순화하면, 내 게임의 몬스터 데이터도 이런 모양으로 설계할 수 있어요.
monster.jsonJSON
// 내 게임용 몬스터 한 마리의 정보 카드 (PokéAPI 구조 참고)
{
  "name": "pikachu",        // 이름
  "types": ["electric"],     // 타입(상성 계산에 사용)
  "hp": 35,                  // 체력
  "attack": 55,              // 공격력
  "speed": 90,               // 누가 먼저 때릴지 결정
  "sprite": "img/pikachu.png" // 화면에 띄울 그림 경로
}
  • types는 배열이에요. 한 몬스터가 두 타입을 가질 수 있어서 ["fire","flying"] 처럼 여러 개를 담을 수 있게 둡니다.
  • speed는 배틀에서 "누가 먼저 공격하는가"를 정할 때 비교하는 값이에요. 이게 없으면 턴 순서를 못 정해요.
  • sprite는 이미지 파일의 위치예요. 실제 포켓몬 이미지 대신 내가 만든/무료 에셋 그림 경로를 넣으면 저작권 문제를 피할 수 있어요.

포켓몬 배틀 로직 (GitHub)

"③ 턴제 배틀"을 만들 때 참고하는 실제 소스코드들이에요. 타입 상성·기술 처리·데미지 계산 같은 복잡한 부분의 정답을 볼 수 있어요. 단, 코드가 어려워서 처음엔 "흐름만" 보고, 뜯어보는 건 나중으로 미루는 걸 권해요.

🔁먼저 "턴제 배틀 흐름"부터 머리에 넣기

코드를 보기 전에 아래 흐름을 알고 보면 훨씬 쉽게 읽혀요.

1단계속도(speed) 비교 → 누가 먼저 공격할지 결정
▼
2단계공격 측 기술 선택 → 타입 상성 계산(2배/0.5배 등)
▼
3단계데미지 계산 → 상대 HP 감소 → HP 바 갱신
▼
4단계쓰러졌는지 확인 → 아니면 반대로 상대 턴 반복

License & checklist

⚠️ 저작권 주의 · 최종 체크리스트

이 부분이 어쩌면 가장 중요해요. 포트폴리오로 공개할 게임이라면 "남의 그림·소리·데이터를 그대로 넣어서 배포"하면 문제가 될 수 있거든요. 라이선스(사용 조건) 개념만 짧게 잡고 가요.

라이선스 한눈에 — 뭘 조심해야 하나
구분예시배포 게임에 써도?
CC0 (퍼블릭 도메인)OpenGameArt 일부, Kenney✅ 출처 없이도 자유
CC-BYOpenGameArt 일부✅ 단, 출처 표기 필수
유료 에셋Aseprite로 산 팩, CraftPix 유료✅ 구매 조건 안에서
실제 포켓몬/롤 IP공식 스프라이트, Data Dragon 에셋❌ 연습용만, 배포 금지
커뮤니티 자료PokeCommunity 등⚠️ 라이선스 불명확 → 조심
🚫가장 흔한 실수

"PokéAPI에서 진짜 포켓몬 이미지를 받아 내 게임에 넣고 itch.io에 공개" → 이게 위험해요. 데이터 구조와 코드 방식은 마음껏 참고하되, 실제 포켓몬 그림·이름·음악은 내가 만든 것이나 무료 에셋(4번 섹션)으로 교체해서 배포하세요.

📋지금 단계 체크리스트
  • Node.js 설치 (1순위)
  • VS Code 설치
  • GitHub 계정 만들기 + Git 설치
  • Phaser 프로젝트 만들고 첫 화면 띄워보기
  • Kenney/itch.io에서 프로토타입용 무료 에셋 1세트 받기
  • PokéAPI 구조 보고 내 monster.json 설계해보기
  • 맵 만들 차례 되면 Tiled 설치
  • 에셋 받을 때마다 라이선스(CC0/CC-BY/유료) 메모해두기

🎯 한 장 요약 — 이것만 기억하면 돼요

  • 지금 필요한 건 5개뿐: Node.js · VS Code · Git/GitHub · Phaser · (곧) Tiled. 나머지는 사전처럼 필요할 때 꺼내 보기.
  • 그림이 막히면 Kenney → itch.io(무료 monster/pixel 태그) → OpenGameArt 순으로.
  • 포켓몬 데이터 설계는 PokéAPI 구조를 정답지로, 배틀 로직은 Pokémon Showdown "흐름"부터.
  • 화면 구성 막히면 Interface In Game · Game UI Database에서 실제 게임 UI 구경.
  • 저작권: 코드·구조는 참고 OK, 실제 포켓몬 그림·이름·소리는 배포 게임에서 무료 에셋으로 교체.
