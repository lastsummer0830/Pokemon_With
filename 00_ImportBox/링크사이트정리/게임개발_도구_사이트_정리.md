# 게임 개발 도구와 사이트 정리

> 정리 기준: **게임 개발 연습**, **포트폴리오 제작**, **초보자가 실제로 설치하고 써볼 수 있는지**를 기준으로 정리했습니다.  
> 원본 파일 `같이 쓰는 프로그램들.txt`에 있던 `Godot`, `Visual Studio Code`, `Aseprite`, `LibreSprite`, `Tiled`를 핵심 조합으로 삼았습니다.  
> 가격, 무료 범위, 라이선스는 시간이 지나면 바뀔 수 있으므로 실제 사용 전에는 반드시 공식 페이지를 한 번 더 확인하는 것이 좋습니다.

---

## 전체 요약

게임 개발을 처음 연습하면서 포트폴리오까지 생각한다면, 처음부터 너무 큰 엔진이나 복잡한 툴을 전부 쓰려고 하기보다 아래 흐름으로 시작하는 것이 좋습니다.

```text
1. Godot으로 작은 2D 게임 만들기
   ↓
2. Visual Studio Code로 코드 편집과 Git 관리 익히기
   ↓
3. Aseprite 또는 LibreSprite로 도트 캐릭터/아이템 만들기
   ↓
4. Tiled로 타일맵과 스테이지 구성하기
   ↓
5. itch.io에 플레이 가능한 빌드 올리기
   ↓
6. GitHub에 소스 코드와 README 정리하기
   ↓
7. 포트폴리오 페이지나 GitHub Pages에 결과물 연결하기
```

핵심은 **게임을 완성했다는 결과물**과 **어떤 도구를 왜 썼는지 설명할 수 있는 기록**을 같이 남기는 것입니다.

# 1. Godot

## 링크

- 공식 사이트: <https://godotengine.org/>
- 다운로드: <https://godotengine.org/download/>
- 라이선스: <https://godotengine.org/license/>
- Asset Library: <https://godotengine.org/asset-library/asset>

## 무엇을 하는 프로그램인가?

`Godot`은 게임을 만드는 **게임 엔진**입니다.

게임 엔진은 캐릭터 이동, 충돌 처리, 화면 출력, 애니메이션, 사운드, UI, 파일 저장 같은 기능을 직접 처음부터 만들지 않도록 도와주는 프로그램입니다.

쉽게 말하면:

```text
내가 게임 규칙과 화면 구성을 만들면,
Godot이 게임 실행에 필요한 기본 시스템을 도와주는 도구
```

입니다.

## 어떤 게임에 잘 맞는가?

- 2D 플랫포머
- 탑다운 RPG
- 퍼즐 게임
- 간단한 액션 게임
- 카드/보드게임 스타일 게임
- 작은 3D 게임 프로토타입

특히 처음 게임 개발을 연습할 때는 `2D 게임`으로 시작하기 좋습니다. 3D보다 화면 구조와 충돌을 이해하기 쉽고, 포트폴리오용 작은 결과물을 만들기도 좋습니다.

## 어떻게 다운로드하는가?

1. <https://godotengine.org/download/> 접속
2. 운영체제에 맞는 버전 선택
3. Windows라면 보통 압축 파일을 내려받아 압축 해제
4. `Godot` 실행 파일을 실행
5. 새 프로젝트 생성 후 작업 시작

Godot은 보통 별도 설치 과정이 복잡하지 않은 편입니다. 압축을 풀고 실행하는 방식이라 처음 써보기 쉽습니다.

## 무료인가?

무료입니다.

Godot Engine은 공식적으로 **MIT License** 기반의 무료 오픈소스 소프트웨어입니다. 공식 라이선스 설명에 따르면 Godot은 어떤 목적으로도 사용할 수 있고, Godot으로 만든 게임의 저작권은 기본적으로 만든 사람에게 있습니다.

다만 Godot 엔진 자체를 게임과 함께 배포하는 경우에는 Godot의 저작권 고지와 라이선스 문구를 포함해야 합니다. 포트폴리오용 작은 게임이라도 `README`나 크레딧에 Godot 사용 사실을 적어두면 좋습니다.

## 포트폴리오에 어떻게 쓰면 좋은가?

Godot으로 포트폴리오를 만들 때는 단순히 “게임 만들었습니다”보다 아래 내용을 같이 정리하는 것이 좋습니다.

- 게임 장르
- 핵심 조작 방법
- 사용한 Godot 버전
- 구현한 기능
- 직접 만든 에셋과 외부 에셋 구분
- 플레이 영상 또는 GIF
- 다운로드 링크 또는 itch.io 링크
- GitHub 저장소 링크

예시:

```text
Godot 4 기반 2D 탑다운 미니 RPG
- 플레이어 이동
- NPC 대화
- 아이템 획득
- 간단한 전투
- Tiled 맵 연동
- 직접 제작한 도트 캐릭터 사용
```

## 초보자 주의점

- 처음부터 대형 RPG나 온라인 게임을 만들려고 하지 말 것
- 2D 이동, 충돌, 씬 전환, UI부터 익힐 것
- 에셋을 가져다 쓸 때는 라이선스를 반드시 기록할 것
- Godot 버전이 다르면 튜토리얼 내용이 안 맞을 수 있으므로 버전을 확인할 것

# 2. Visual Studio Code

## 링크

- 공식 다운로드: <https://code.visualstudio.com/Download>
- 라이선스: <https://code.visualstudio.com/License/>

## 무엇을 하는 프로그램인가?

`Visual Studio Code`, 줄여서 `VS Code`는 코드를 작성하고 편집하는 프로그램입니다.

Godot 안에도 스크립트 편집기가 있지만, 프로젝트가 조금 커지면 VS Code를 같이 쓰는 것이 편합니다.

VS Code에서 할 수 있는 일은 다음과 같습니다.

- 코드 작성
- 폴더 구조 확인
- Git 변경 사항 확인
- 확장 프로그램 설치
- Markdown 문서 작성
- 터미널 실행
- 검색과 파일 이동

## Godot과 같이 쓰는 이유

Godot의 스크립트 파일은 보통 `GDScript`를 사용합니다. VS Code에 Godot 관련 확장을 설치하면 코드 자동완성, 문법 강조, 파일 관리가 더 편해질 수 있습니다.

또한 포트폴리오를 만들 때는 게임 파일만 중요한 것이 아니라 `README.md`, 작업 기록, 실행 방법 설명도 중요합니다. 이런 문서 작업에는 VS Code가 편합니다.

## 어떻게 다운로드하는가?

1. <https://code.visualstudio.com/Download> 접속
2. Windows, macOS, Linux 중 운영체제 선택
3. Windows라면 보통 `User Installer` 또는 `System Installer` 선택
4. 설치 후 실행
5. 필요한 확장 프로그램 설치

게임 개발 연습용으로 설치하면 좋은 확장 예시는 다음과 같습니다.

- Godot Tools
- GitLens
- Markdown All in One
- vscode-icons 또는 Material Icon Theme

## 무료인가?

무료로 사용할 수 있습니다.

공식 다운로드 페이지에서도 `Free and built on open source`라고 안내합니다. 다만 Microsoft에서 배포하는 VS Code 제품 자체는 Microsoft 소프트웨어 라이선스를 따르고, 확장 프로그램은 각각 별도 라이선스를 가질 수 있습니다.

일반적인 개인 학습, 포트폴리오, 프로젝트 코드 작성에는 무료로 사용해도 됩니다.

## 포트폴리오에 어떻게 쓰면 좋은가?

VS Code 자체가 포트폴리오 결과물이 되는 것은 아니지만, 아래 자료를 정리하는 데 중요합니다.

- `README.md`
- 설치 방법
- 실행 방법
- 조작법
- 사용 기술
- 구현 기능 목록
- Git commit 기록

특히 GitHub에 올릴 때 `README.md`가 정리되어 있으면 포트폴리오 완성도가 훨씬 좋아 보입니다.

## 초보자 주의점

- VS Code와 Visual Studio는 다른 프로그램입니다.
- Godot 2D 입문에는 VS Code만으로도 충분한 경우가 많습니다.
- Unity C# 개발을 깊게 할 때는 Visual Studio Community가 더 편할 수 있습니다.

# 3. Aseprite

## 링크

- 공식 사이트: <https://www.aseprite.org/>
- 다운로드/구매: <https://www.aseprite.org/download/>

## 무엇을 하는 프로그램인가?

`Aseprite`는 픽셀 아트와 스프라이트 애니메이션을 만드는 프로그램입니다.

게임에서 캐릭터가 걷는 모습, 공격 모션, 아이템 아이콘, 배경 타일 같은 도트 그래픽을 만들 때 자주 사용합니다.

## 어떤 작업에 쓰는가?

- 캐릭터 도트 그림
- 걷기 애니메이션
- 공격 애니메이션
- 아이템 아이콘
- UI 아이콘
- 타일셋 제작
- 스프라이트 시트 내보내기

## 어떻게 다운로드하는가?

1. <https://www.aseprite.org/download/> 접속
2. 공식 구매 페이지에서 라이선스 구매
3. Windows, macOS, Ubuntu 버전 중 선택
4. 설치 후 실행

Aseprite는 Steam에서도 구매할 수 있습니다. 공식 페이지에서는 Windows, macOS, Ubuntu용 다운로드와 Steam key 제공을 안내합니다.

## 무료인가?

공식 빌드는 유료입니다.

Aseprite는 픽셀 아트 도구로 매우 유명하지만, 공식 다운로드는 구매가 필요합니다. 소스 코드는 공개되어 있으나, 일반 사용자가 편하게 설치해서 쓰는 공식 빌드는 유료라고 이해하면 됩니다.

무료 대체 프로그램으로는 아래의 `LibreSprite`가 있습니다.

## 포트폴리오에 어떻게 쓰면 좋은가?

Aseprite로 만든 자료는 포트폴리오에서 강점이 될 수 있습니다.

예시:

- 직접 만든 캐릭터 스프라이트
- 걷기/점프/공격 애니메이션 GIF
- 아이템 아이콘 모음
- 타일셋 이미지
- 실제 게임에 적용한 화면 캡처

포트폴리오에서는 “직접 그린 것”과 “외부 에셋을 사용한 것”을 구분해서 적는 것이 중요합니다.

## 초보자 주의점

- 처음부터 큰 캐릭터를 그리기보다 16x16, 32x32 크기부터 시작하기
- 색을 너무 많이 쓰지 않기
- 한 캐릭터의 정면, 측면, 걷기 모션부터 연습하기
- 작업 파일과 내보낸 PNG를 따로 관리하기

# 4. LibreSprite

## 링크

- GitHub 저장소: <https://github.com/LibreSprite/LibreSprite>
- 공식 홈페이지: <https://libresprite.github.io/>
- 최신 릴리즈: <https://github.com/LibreSprite/LibreSprite/releases/latest>

## 무엇을 하는 프로그램인가?

`LibreSprite`는 무료 오픈소스 픽셀 아트 프로그램입니다.

공식 설명에 따르면 LibreSprite는 스프라이트를 만들고 애니메이션을 만들 수 있는 무료 오픈소스 프로그램이며, Aseprite의 과거 GPLv2 버전에서 갈라져 나온 프로젝트입니다.

## 어떤 작업에 쓰는가?

- 도트 캐릭터 제작
- 스프라이트 애니메이션 제작
- 팔레트 관리
- 레이어와 프레임 기반 작업
- 간단한 타일/패턴 작업

## 어떻게 다운로드하는가?

1. <https://github.com/LibreSprite/LibreSprite/releases/latest> 접속
2. Windows, macOS, Linux, Android 중 필요한 파일 확인
3. 운영체제에 맞는 릴리즈 파일 다운로드
4. 압축 해제 또는 설치 후 실행

## 무료인가?

무료입니다.

LibreSprite는 GitHub 기준 `GNU General Public License v2.0`, 즉 `GPL-2.0` 라이선스를 사용합니다. 프로그램 자체는 무료 오픈소스입니다.

일반적으로 LibreSprite로 만든 그림 자체는 내가 만든 창작물이지만, 프로그램을 수정해서 배포하는 경우에는 GPL 조건을 확인해야 합니다.

## Aseprite와 비교하면?

| 구분 | Aseprite | LibreSprite |
|---|---|---|
| 가격 | 공식 빌드 유료 | 무료 |
| 사용 편의성 | 매우 좋음 | 기본 기능 중심 |
| 업데이트/생태계 | 활발하고 자료 많음 | 상대적으로 작음 |
| 추천 상황 | 계속 픽셀 아트를 할 생각이 있을 때 | 무료로 먼저 연습할 때 |

## 포트폴리오에 어떻게 쓰면 좋은가?

무료 도구로 직접 만든 픽셀 아트를 게임에 적용했다는 점을 보여줄 수 있습니다.

예시:

```text
LibreSprite로 캐릭터 4방향 이동 스프라이트 제작
Godot에서 AnimatedSprite2D로 적용
Tiled 맵 위에서 캐릭터 이동 구현
```

# 5. Tiled

## 링크

- 공식 사이트: <https://www.mapeditor.org/>

## 무엇을 하는 프로그램인가?

`Tiled`는 타일맵과 레벨을 만드는 프로그램입니다.

타일맵은 작은 이미지 조각을 격자처럼 배치해서 맵을 만드는 방식입니다. 예를 들어 2D RPG의 마을, 던전, 숲, 길, 벽, 바닥을 만들 때 자주 사용합니다.

## 어떤 작업에 쓰는가?

- 2D RPG 맵 제작
- 플랫포머 스테이지 제작
- 던전 구조 제작
- 충돌 영역 표시
- 오브젝트 배치
- JSON, TMX 등으로 맵 데이터 내보내기

공식 사이트에서는 Tiled를 무료 오픈소스이며 유연한 레벨 에디터라고 소개합니다. 타일 레이어, 오브젝트 레이어, 큰 월드 편집, JavaScript 확장, 여러 포맷 내보내기 등을 지원합니다.

## 어떻게 다운로드하는가?

1. <https://www.mapeditor.org/> 접속
2. Download 메뉴로 이동
3. 운영체제에 맞는 설치 파일 다운로드
4. 설치 후 새 맵 생성
5. 타일셋 이미지를 불러와 맵 제작

## 무료인가?

무료 오픈소스입니다.

다만 개발 지속을 위해 후원이나 유료 스토어 배포가 함께 있을 수 있습니다. 공식 사이트에서 제공하는 기본 사용은 무료 오픈소스 도구로 보면 됩니다.

## Godot과 같이 쓰는 방법

Tiled에서 만든 맵을 Godot에서 쓰려면 보통 아래 흐름을 사용합니다.

```text
1. Aseprite 또는 LibreSprite로 타일셋 이미지 제작
2. Tiled에서 타일셋 불러오기
3. Tiled에서 맵 배치
4. TMX 또는 JSON으로 저장/내보내기
5. Godot에서 플러그인이나 변환 방식으로 가져오기
```

처음에는 Godot 자체의 TileMap 기능만 써도 됩니다. Tiled는 맵이 커지거나 외부 맵 에디터로 분리하고 싶을 때 쓰면 좋습니다.

## 포트폴리오에 어떻게 쓰면 좋은가?

Tiled를 쓰면 “맵 설계”를 보여주기 좋습니다.

- 스테이지 구조
- 장애물 배치
- 길 찾기 동선
- 전투 구역
- NPC 위치
- 아이템 위치

포트폴리오에는 완성 화면뿐 아니라 Tiled에서 작업한 맵 화면도 같이 캡처하면 좋습니다.

### 무엇을 하는 곳인가?

포트폴리오에서는 단순히 게임 실행 파일만 올리는 것보다, GitHub에 소스 코드와 README를 정리해두는 것이 좋습니다.

### 무료인가?

GitHub Free 플랜은 무료이며, 공식 가격 페이지 기준으로 공개/비공개 저장소를 만들 수 있습니다. 팀 기능, 기업 기능, 저장 용량, Actions 사용량 등은 플랜별 제한이 있습니다.

### 포트폴리오에 어떻게 쓰면 좋은가?

게임 프로젝트 저장소에는 최소한 아래 내용을 넣는 것이 좋습니다.

```text
README.md
- 게임 소개
- 실행 방법
- 조작법
- 사용 기술
- 구현 기능
- 스크린샷
- 플레이 링크
- 사용 에셋 출처
```

## GitHub Pages

### 링크

- 공식 문서: <https://docs.github.com/en/pages/getting-started-with-github-pages/about-github-pages>

### 무엇을 하는 곳인가?

`GitHub Pages`는 GitHub 저장소의 HTML, CSS, JavaScript 파일을 웹사이트로 공개할 수 있는 정적 사이트 호스팅 기능입니다.

공식 문서에서는 개인/조직 사이트와 프로젝트 사이트를 만들 수 있다고 설명합니다.

### 게임 포트폴리오에서 어떻게 쓰는가?

- 자기소개 페이지
- 프로젝트 모음 페이지
- 게임 스크린샷 모음
- itch.io 링크 연결
- GitHub 저장소 링크 연결
- 개발 회고 정리

# 11. 에셋 마켓과 라이브러리

## Godot Asset Library

Godot용 플러그인, 템플릿, 데모, 스크립트, 툴을 찾을 수 있는 공식 에셋 라이브러리입니다.

검색 화면에서 라이선스, Godot 버전, 카테고리 등을 확인할 수 있습니다.

## Unity Asset Store

### 링크

- Asset Store: <https://assetstore.unity.com/>
- Unity 매뉴얼: <https://docs.unity3d.com/Manual/AssetStore.html>

### 무엇을 하는 곳인가?

Unity Asset Store는 Unity Technologies와 커뮤니티가 만든 무료/유료 에셋을 제공하는 마켓입니다.

공식 매뉴얼 기준으로 텍스처, 모델, 애니메이션, 프로젝트 예제, 튜토리얼, Unity Editor 확장 등이 포함됩니다.

### 주의점

- 무료 에셋과 유료 에셋이 섞여 있음
- 에셋별 라이선스와 사용 조건 확인
- 포트폴리오에서는 직접 구현한 부분과 에셋 사용 부분을 구분

## Fab

### 링크

- <https://www.fab.com/>

### 무엇을 하는 곳인가?

`Fab`은 Epic Games 계열의 에셋 마켓입니다. Unreal Engine, 3D 모델, 환경, 애니메이션, 머티리얼 등 다양한 에셋을 찾을 수 있습니다.

무료 콘텐츠, 할인 콘텐츠, 유료 콘텐츠가 섞여 있습니다.

### 주의점

- 무료라고 해도 라이선스와 사용 범위를 확인
- Unreal 프로젝트에 바로 쓰기 좋은 에셋이 많음
- 다른 엔진에서 쓸 때는 파일 포맷과 라이선스 확인

## 사용 에셋

- 에셋명: Platformer Pack
- 제작자: Kenney

# 18. 개인 메모

현재 원본 TXT의 조합은 꽤 좋습니다.

```text
게임 제작: Godot
코드 편집: Visual Studio Code
도트 그림: Aseprite
무료 도트: LibreSprite
맵 제작: Tiled
```

여기에 포트폴리오까지 생각하면 아래 3개를 추가로 꼭 챙기면 좋습니다.

```text
배포: itch.io
코드 공개: GitHub
무료 에셋: Kenney
```

이 조합이면 처음에는 비용을 거의 들이지 않고도 작은 2D 게임을 만들고, 공개하고, 포트폴리오로 정리하는 흐름까지 연습할 수 있습니다.
