처음 보는 사람을 위한 추천 사용 순서

AI Agent 개발환경을 처음 만들 때의 큰 순서입니다. 실제 명령은 각 도구 공식 문서에서 최신 버전을 확인하세요.

AI Agent 환경 구성 흐름Guide
1. Node.js 설치  // npm 기반 도구 실행 준비
2. OpenCode 설치/연결  // 프로젝트 폴더에서 AI Agent 사용
3. AGENTS.md 작성  // Agent에게 작업 규칙 전달
4. Obsidian + Web Clipper  // 자료 수집과 지식 저장
5. NotebookLM / Hugging Face / Exa  // 자료 기반 학습, 모델 탐색, 검색 연결
6. Supabase / Neo4j / Slack / Tailscale  // 실제 서비스와 자동화 확장
  • 설치형 도구는 OS와 버전에 따라 명령어가 바뀔 수 있으므로, 카드의 공식 링크에서 최신 설치법을 확인합니다.
  • 로그인/권한이 필요한 내부 페이지는 “개념 확인용”으로 보고, 공개 공식 문서를 우선합니다.
  • 처음부터 모든 도구를 설치하지 말고, 목적별로 필요한 것만 선택합니다.

상황별 빠른 선택표

AI Agent 개발환경 만들기Node.js → OpenCode → AGENTS.md실행환경, Agent 도구, 작업 규칙이 기본 3종 세트입니다.
웹 자료를 지식으로 저장하기Obsidian Web Clipper → PKM Course웹페이지를 vault에 넣고, 나중에 AI가 읽을 수 있는 자료로 축적합니다.
모델/성능 비교하기Hugging Face → Artificial Analysis모델 후보를 찾고, 속도/가격/성능 지표를 비교합니다.
Agent에 검색 기능 붙이기Exa일반 검색보다 agent/research 워크플로우에 맞춘 API를 제공합니다.
서비스 백엔드 빠르게 만들기SupabaseDB, Auth, Storage, Realtime을 한 번에 준비할 수 있습니다.
지식 그래프/GraphRAG 해보기Neo4j → Graphify그래프 DB와 문서/코드 그래프화를 연결해 볼 수 있습니다.
발표/시각 자료 만들기Stitch → Penpot → redhat-slide-skillAI UI 생성, 디자인 협업, 슬라이드 제작을 이어갈 수 있습니다.
