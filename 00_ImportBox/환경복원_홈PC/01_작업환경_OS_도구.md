> 2026-06-24 학원 PC에서 실제로 깔려 있던 환경. 집 PC도 이만큼 맞추면 동일하게 돌아간다.
> 버전은 "이 이상이면 OK" 기준(굳이 똑같을 필요는 없지만, 의심스러우면 이 버전으로).

## 1. 어떤 환경에서 작업하나 (전체 그림)

- **Windows + WSL2** 조합. 실제 코드/명령은 **WSL2 안의 Ubuntu**에서 돈다.
- 프로젝트 파일은 **Windows의 D드라이브**(`D:\...` = WSL에선 `/mnt/d/...`)에 있다.
- 게임 자체는 두 가지로 실행:
  - **개발 중**: WSL에서 `npm run dev` → Vite 개발서버(http://localhost:5180) → Windows 브라우저로 봄. (`localhost`는 윈도우 `wslrelay.exe`가 WSL로 포워딩)
  - **완성본**: Electron으로 구운 **`Pokemon With.exe`**(Windows 네이티브 실행파일)를 더블클릭.
- 이미지/맵 가공은 WSL의 **Python(Pillow/numpy/rubymarshal)** + **ffmpeg**로 한다.

### 🔴 D드라이브 + WSL 치명적 함정 (꼭 기억)
WSL에서 윈도우 드라이브(`/mnt/d`)는 **파일 변경 감지(inotify)가 안 된다.** 그래서 `myPokemon_AJ/vite.config.ts`에 **`server.watch.usePolling: true`**(interval 200)가 들어 있다. **절대 지우지 마라.** 없으면 코드를 고쳐도 브라우저에 반영이 안 되는데 서버 curl로는 새 코드가 보여서 디버깅이 미궁에 빠진다. 화면 안 바뀌면 브라우저 **Ctrl+Shift+R**.

## 4. Python 패키지 (이미지·맵 가공에 필수)

학원 PC 실측: **Pillow 12.2.0, numpy 2.4.6, rubymarshal(설치됨)**.

Ubuntu 24.04는 시스템 파이썬이 **PEP 668(externally-managed)** 로 막혀 있어서 그냥 `pip install`이 안 된다. 학원 PC에선 이렇게 풀었다:

```bash
# 방법 A: --user --break-system-packages (학원 PC에서 쓴 방식)
python3 -m pip install --user --break-system-packages -q numpy Pillow rubymarshal

# pip 자체가 없으면 먼저:
curl -sS -o get-pip.py https://bootstrap.pypa.io/get-pip.py
python3 get-pip.py --user --break-system-packages -q
```

> 더 깔끔하게 하려면 venv를 써도 된다(`python3 -m venv .venv && source .venv/bin/activate && pip install numpy Pillow rubymarshal`). 단 `tools/ar-map/*.py`는 시스템 python3로 바로 돌렸으니, venv를 쓰면 실행할 때 그 venv의 python으로 돌려야 한다.

| 패키지 | 용도 |
|---|---|
| **Pillow (PIL)** | 스프라이트 crop/resize(NEAREST)·몽타주 생성·맵 타일 합성 |
| **numpy** | 이미지 배열 처리(팔레트/픽셀 연산) |
| **rubymarshal** | ⭐ **어나더레드 `.rxdata`(Ruby Marshal) 파싱** — 맵 데이터 읽기 (`03_AR맵_...` 참고) |

## 6. Electron / 데스크톱 빌드 환경 메모

- 진입점: `myPokemon_AJ/electron/main.cjs`.
- WSLg에서 WebGL 막히면 `main.cjs`의 `ignore-gpu-blocklist` / `enable-unsafe-swiftshader` 플래그가 풀어준다(실제 윈도우엔 영향 없음). WSL 실행 시 `--no-sandbox` 필요할 수 있음.
- **exe 굽기(중요):** WSL에서 그냥 `electron-builder` 돌리면 **리눅스 타겟**을 굽는다(헛수고). 반드시:
  ```bash
  cd myPokemon_AJ
  npm run build && npx electron-builder --win --x64
  # 산출물: build_win/win-unpacked/Pokemon With.exe
  ```
- 굽기 전 게임이 켜져 있으면 dll 잠김으로 실패 → PowerShell로 종료:
  `powershell.exe -NoProfile -Command "Get-Process -Name 'Pokemon With' -ErrorAction SilentlyContinue | Stop-Process -Force"`
- `npm install` 때 Electron 본체(~216MB) 504 뜨면: `ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/ npm install`.
