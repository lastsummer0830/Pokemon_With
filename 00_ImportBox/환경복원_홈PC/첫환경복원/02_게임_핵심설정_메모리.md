# 02. 게임 핵심 설정 + Claude 메모리 (내가 주요하게 박아둔 것들)

> 이게 "이 게임은 이런 게임이고, Claude는 이렇게 행동해야 한다"는 **확정 규칙 모음**이다.
> 두 군데에 박혀 있다: ① 프로젝트 안 `myPokemon_AJ/AGENTS.md`(코드와 같이 git로 따라옴) ② Claude **메모리**(repo 밖 → `복사본/claude_memory/`로 챙김).
> 아래는 그 메모리들의 내용을 사람이 읽기 쉽게 풀어 쓴 것. **원본 md는 `복사본/claude_memory/`에 그대로 있다.**

## A. 게임 확정 설정 (절대 변경 금지 — 사용자 직접 지정)

1. **게임 정체성**: 2D 탑다운 포켓몬 팬게임(HGSS/하트골드 감성 맵 이동 + 턴제 배틀). 목표 퀄리티 = 팬게임 **"어나더 레드(Another Red)" 수준**. 어설픈 프로토타입 금지.
2. **차별점(내 색)**: **집 꾸미기 → 포켓몬 컨디션 ↑ → 배틀 보너스**. (예: 불꽃 포켓몬을 벽난로 옆에서 재우면 컨디션↑ → 배틀에서 강해짐)
3. **화풍 = 픽셀(도트) 전용.** 매끈한 일러스트(공식 아트워크, PokeAPI HOME 512 등) **인게임 금지**. "고화질/최품질"이라고 해도 = **"최고 품질의 픽셀"**을 뜻함. (메모리 `dot-style-only` — 이거 어겨서 여러 번 혼남)
4. **주인공 확정**: **남자 = 1세대 RED**, **여자 = 4세대 DAWN**. 전부 픽셀. (메모리 `protagonist-choice`)
5. **대화 UI**: 말하는 사람 있는 대사는 대화박스 위 **이름창** 붙임. 나레이션/상황설명은 이름창 없이 박스만. (`IntroScene`의 `setSpeaker`/`applySpeaker`)
6. **인트로 진행 순서**: 성별 선택 → 이름 입력. 각각 전용 화면(오버레이 아님). 이름 화면엔 선택한 성별 카드 표시, 확정 시 예/아니오 확인.

> 위 1~6은 `myPokemon_AJ/AGENTS.md` §1, §1.5에 원본이 있다. **AGENTS.md가 규칙의 단일 원본**이고, 메모리는 그걸 보강/강조한다.

## C. 메모리 복원 방법 (집 PC) — ⚠️ 슬러그 주의

Claude 메모리는 repo 밖, 홈 디렉토리에 있다:
```
/home/<유저>/.claude/projects/<프로젝트경로-슬러그>/memory/
```
`<프로젝트경로-슬러그>`는 **프로젝트 절대경로를 변환**해 자동 생성된다. 학원 PC에선:
```
/mnt/d/dev/AJ_Proj/vcPortfolio_AJ  →  -mnt-d-dev-AJ-Proj-vcPortfolio-AJ
```

**집 PC에서 프로젝트 경로가 같으면** 슬러그도 같아서 그냥 같은 위치에 복사하면 된다.
**경로가 다르면** 슬러그가 달라진다. 그땐:
1. 집 PC에서 이 프로젝트 폴더로 Claude Code를 한 번 띄운다(메모리 폴더가 자동 생성됨).
2. 실제 생성된 `~/.claude/projects/<...>/memory/` 경로를 확인한다.
3. 그 폴더에 `복사본/claude_memory/*.md`(MEMORY.md 포함)를 복사한다.

> `MEMORY.md`는 인덱스(한 줄 요약 목록), 나머지 `*.md`가 실제 메모리 본문이다. 둘 다 넣어야 한다.

복사할 파일(이 폴더 `복사본/claude_memory/`에 다 있음):
`MEMORY.md`, `mypokemon-project.md`, `dont-over-ask.md`, `dot-style-only.md`, `protagonist-choice.md`, `pick-folder-workflow.md`, `launch-via-bat-only.md`, `user-fullstack.md`, `ar-map-rxdata-pipeline.md`
