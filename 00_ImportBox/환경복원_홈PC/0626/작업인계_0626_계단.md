## 3. ⚠️ 가장 마지막에 한 작업 = **여기서 멈춤 (이어서 해야 함)**
사용자는 **dev서버(bat)가 아니라 패키징된 `Pokemon With.exe`로 게임을 실행**한다(중요!). exe는 자기 안에 구워진 `app.asar`의 코드로 돌아가므로 소스 수정이 자동 반영 안 됨.
- WSL엔 **wine이 없어 `electron-builder --win`(정식 exe 빌드)이 불가** → **asar 핫스왑** 방식으로 exe 안 코드를 갈아끼우는 중이었음:
  1. `npm run build` (dist 새로 구움 — 완료, 번들 `index-3JoCQmPO.js`, 대화상자 수정 포함)
  2. `npx asar extract` 로 기존 app.asar 풀고 → `dist`만 새 걸로 교체 → `npx asar pack` 로 다시 패킹 (완료)
  3. 패킹 결과 = **`build_win/win-unpacked/resources/app.asar.new`** (17:46, 최신=대화상자 수정 포함)
  4. **`app.asar.new` → `app.asar` 교체에서 `Permission denied`로 실패** ← **게임(exe)이 켜져 있어서 파일 잠김.**
- **현재 exe 상태:** `app.asar`(17:22)는 **계단/충돌 새 격자는 들어가 있으나 대화상자 수정은 없음.** 최신본은 옆에 `app.asar.new`로 대기 중.

## 5. 유의사항 / 특이사항 (집 PC에서 꼭 알아야)
- **실행 방식 혼동 주의:** `게임실행.bat`은 사실 `npm run app`(= **dev 서버 + 소스 실행**)을 돌림. 그런데 **사용자는 패키징된 `Pokemon With.exe`를 직접 실행**한다. 둘은 **다른 코드 경로**(bat=소스 즉시반영 / exe=app.asar에 구운 코드). → 코드 고치면 **exe도 다시 구워야** 사용자한테 반영됨. (메모리 `launch-via-bat-only`는 명칭이 헷갈리는데, 실질 본체는 **exe**.)
- **exe 로드 경로:** `electron/main.cjs`가 프로덕션에서 `../dist/index.html`(app.asar 내부)을 로드. dev에선 `localhost:5180`.
- **wine 없으면 WSL에서 win exe 못 굼.** 집 PC가 네이티브 Windows면 `npm run app:build`로 정석 빌드 가능.
- **`/mnt/d`(D드라이브) inotify 안 됨** → `vite.config.ts`의 `server.watch.usePolling:true` 절대 삭제 금지. 코드 고쳐도 화면 안 바뀌면 브라우저 `Ctrl+Shift+R`.
- **화풍 = 픽셀(도트) 전용.** 매끈한 일러스트 금지. 남주=1세대 RED, 여주=4세대 DAWN, 라이벌=네모(Nemona, 9세대). 에셋은 손으로 그리지 말고 소스(PokeAPI/AnotherRed)에서.
- 어색한 칸/충돌 발견 시 **추측 말고** §2의 그리드 오버레이(`DEBUG_COLLISION=true`)+playwright 이동 검증으로 잡을 것.
