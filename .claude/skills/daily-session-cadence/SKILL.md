# 세션 케이던스: 계획 → 한 블록 작업 → 검증물 → 일지 → /clear

## 왜 이 사이클인가

긴 세션 = 컨텍스트 누적 = 규칙 망각·품질 저하. `/clear`는 대화를 비우지만, 비워도 되려면 **다음 세션이 이어받을 기록**이 남아 있어야 한다. 이 스킬은 그 기록과 끊는 타이밍을 강제한다.

## 1. 계획 수립 ("오늘 계획" 요청 받으면 맨 먼저)

- 프로젝트 정본부터 읽는다: AJ_Proj → `PORTFOLIO_STATUS.md`+로드맵 / MyWorld → `REBUILD_PLAN.md` / Pokemon → 최근 작업일지 / Acorn → 내 담당 현황 / MyStudy → 요청 자체.
- 오늘 목표를 **세션 블록**으로 쪼갠다. 블록 1개 = 새 세션 하나가 컨텍스트 여유를 갖고 끝낼 분량 (기준: 파일 3~5개 수정, 기능 1개, 문서 1편. 확신 없으면 더 작게).
- 블록마다 4줄: 목표 / 산출물 / 검증 방법 / 위험 1개. 순서는 의존성 순.
- 계획을 보여주고 확인받으면 **블록 1을 바로 시작한다** (계획만 내놓고 멈추지 않는다 — 실행 강제 원칙은 각 워크스페이스 규칙도 함께 적용).

## 2. 작업 — 딱 한 블록만

- 이번 세션은 승인된 블록 하나만. **일찍 끝나도 다음 블록을 시작하지 않는다** — 남는 컨텍스트는 검증·일지 품질에 쓴다.
- 블록 도중 컨텍스트가 무거워지면(파일 덤프·대량 조사 발생) 블록을 쪼개고 중간이라도 종료 절차(§3~5)로 간다.

## 3. 사용자 확인물 제시 (완료 선언의 조건)

- "됐어요"로 끝내지 않는다. 반드시: **바뀐 파일 경로 목록 + 확인 방법** (실행 명령 / 열어볼 파일 / 캡처).
- UI 작업 = 스크린샷, 코드 = 실행·테스트 결과, 문서 = 파일 위치와 봐야 할 섹션.

## 4. 작업일지 (다음 세션의 유일한 기억)

- 위치는 프로젝트 규칙 우선: AJ_Proj → `<MyStudy>/02_MY_NOTES/연동관련/AJ_PROJ/YYYY-MM-DD_AJ_Proj_작업일지.md` / Pokemon → `00_ImportBox/작업일지/MMDD_...` / Acorn → `<MyStudy>/02_MY_NOTES/연동관련/ACORN-E-학습터/YYYY-MM-DD_E학습터_작업일지.md` / MyStudy → CLAUDE.md의 작업일지 규칙 / 규칙 없는 곳은 위치를 물어본다.
- 내용 5가지: ① 한 것(파일 경로까지) ② 검증 결과 ③ 미검증·미결 ④ **다음 세션 시작 지점**(열 파일·첫 명령) ⑤ 남은 블록 목록.
- 같은 날 파일이 있으면 덮지 말고 이어 쓴다.

## 5. 종료 멘트 (매 블록 끝 고정 — 생략 금지)

> 여기서 `/clear` 추천합니다. 다음 세션 첫 프롬프트: **"작업일지 YYYY-MM-DD 읽고 블록 N부터"**

- 같은 날 여러 블록이면 블록마다 이 사이클(§2~5)을 반복한다.
- 계획 요청이 없었어도, 세션이 길어졌다고 판단되면 §3~5를 선제 수행하고 /clear를 권한다.
