# 가방·도감 UI 시안 (4안) — 고르는 법

`_미리보기/전체비교.png` 한 장이면 다 보인다. 화면별 크게 보기:
- `_미리보기/가방_4안.png` · `_미리보기/도감목록_4안.png` · `_미리보기/도감상세_4안.png`

전부 **게임에서 실제 렌더한 스샷**이다. 게임에서 직접 보려면:
`http://localhost:5180` → 타이틀에서 **D** → 목록 아래 **"가방 — A/B/C/D", "도감 — A/B/C/D"** 클릭.

## 네 안의 차이는 **색뿐**이다
그림(프레임·격자·탭·슬라이더·가방·아이콘)은 **Another Red 원본 PNG 그대로**를 쓰고, 배치도 원본 UI 스크립트(`UI_Bag`, `UI_Pokedex_Main/Entry`)의 **실제 좌표를 그대로** 옮겼다. 내가 그린 패널은 하나도 없다.
색만 `tools/ui-pastel.py`로 원본 픽셀을 리컬러한다(픽셀 배치는 1px도 안 건드림 → 언제든 되돌리기·재조정 가능).

| | A안 | B안 | C안 | **D안** |
|---|---|---|---|---|
| 색 | **원본 그대로** (DS 정품 채도) | **파스텔** — 색상 유지 + 채도↓·밝기↑ | **하늘톤** — 타이틀 색감 + 커서 연분홍 | **크림 + 빨강 포인트** |
| 가방 | 포켓별 쨍한 색(주황/분홍…) | 같은 색의 살구·로즈 | 포켓 상관없이 연하늘 | 따뜻한 크림 바탕, 커서만 빨강 |
| 도감 | 빨강 | 연한 로즈 | 연하늘 | 크림 바탕 + AR 특유의 빨강 띠·커서·슬라이더 |
| 느낌 | 원본과 100% 동일 | 눈이 편함 | 가장 차분·타이틀과 통일 | 원본 빨강의 정체성은 남기고 바탕만 부드럽게 |

> C안은 가방 **포켓별 색 구분이 사라진다**(전부 하늘색). D안은 빨강이 포인트로만 남아 구분은 탭 아이콘으로 한다.
> 색조를 더 조정하고 싶으면 `tools/ui-pastel.py`의 채도/밝기 계수와 `SKY_HUE`·`CREAM_HUE`·`RED_HUE`만 바꿔 재생성하면 된다(원본 픽셀은 그대로 두고 매번 다시 뽑는 구조).

## 둘 다(셋 다) 공통으로 이미 되는 것
- 가방: 포켓 3개(회복약/몬스터볼/일반) ←→ 전환, ↑↓ 선택, **회복약은 Enter로 파티에게 사용**(HP·상태이상 회복), 하단 아이템 설명, 7줄 넘으면 슬라이더.
- 도감: 칸토 151칸, 못 본 건 `----------`, 본 것=흰 원 / 잡은 것=몬스터볼. Enter로 상세(스프라이트·분류·타입 아이콘·신장·체중·**한글 도감설명**·발자국).
