# 14_카메라프레이밍 — 야외 화면에 몇 칸이 보이게 할지 (A/B 중 하나 고르기)

**왜 고르나:** "건물이 작아 보인다"의 원인이 개별 건물 크기가 아니라 **카메라 프레이밍**이었다.
- 원본 Another Red = 내부해상도 512×384 / 타일 32px → **창 크기와 무관하게 항상 16×12칸**
- 우리(수정 전) = 타일 64px 고정 + 캔버스가 창 전체 → 창이 클수록 칸이 늘어남(1280창 20칸 / 1920창 30칸)
  → 같은 건물이 원본의 0.8배·0.53배로 보였다.

수정 후엔 **창 크기에서 배율을 역산**해 항상 같은 프레이밍을 유지한다. 다만 우리 창은 16:9,
원본은 4:3이라 **가로·세로를 동시에 16×12로 맞출 수는 없다.** 그래서 둘 중 하나를 고른다.

## A — 가로맞춤 (16 × 9칸)
`A_가로맞춤_16x9칸(건물더큼).png`
- 가로 칸수가 원본과 **정확히 같다**(16칸). 타일 120px(1920 기준) — 건물이 가장 크게 보인다.
- 대신 세로가 9칸뿐이라 원본(12칸)보다 답답하고, 아래에서 다가오는 트레이너·NPC를 늦게 본다.

## B — 세로맞춤 (21.3 × 12칸)
`B_세로맞춤_21x12칸(AR세로동일).png`
- 세로 칸수가 원본과 **정확히 같다**(12칸). 같은 화면높이에서 원본과 타일 픽셀크기도 동일(1080→90px).
- 원본이라면 좌우에 검은 여백이 생길 자리를 맵으로 채운다 → 우리집이 충분히 크면서 옆집·연구소까지 보인다.
